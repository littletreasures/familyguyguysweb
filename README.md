# Family Guy Guys — Podcast Reviews Project

A comprehensive web application and admin pipeline for managing and displaying episode reviews for the "Family Guy Guys" podcast. This project consists of a Vite React frontend for users to browse and read reviews, and Python-based admin tools to generate, edit, and publish reviews directly to a Supabase database.

## Project Directory Mapping

The repository is structured as follows:

*   **`admin-tools/`** — Python admin utilities, including a Streamlit web app for review generation and database upserting, CLI tools for metadata fetching, and LLM-based transcript review generators.
*   **`src/`** — React frontend source code built on Vite, including components, pages, routing, and database integrations.
*   **`public/`** — Static assets served directly by the Vite React frontend.
*   **`fixed-assets/`** — Source or reference asset files for the project.
*   **`dist/`** — Production build output folder generated by Vite.
*   **`package.json`** — Node.js dependencies and script definitions for the frontend.
*   **`vite.config.ts`** — Vite build configuration.
*   **`wrangler.jsonc`** — Deployment configuration for Cloudflare Pages/Workers.

## Admin Tools Setup & Run Guide

The admin tools are Python utilities for fetching episode metadata and generating episode reviews from Riverside transcripts using LLMs (Gemini, OpenAI, or Anthropic).

### Prerequisites

*   Python 3.9 or higher
*   pip (Python package manager)

### Installation

1.  Navigate to the `admin-tools` directory:
    ```bash
    cd admin-tools
    ```
2.  Install the required Python packages:
    ```bash
    pip install -r requirements.txt
    ```

### Environment Configuration

Create a `.env` file inside the `admin-tools` directory. You can copy the provided example configuration file:

```bash
cp .env.example .env
```

Open `.env` and fill in the appropriate API keys:

```env
# Selected LLM Provider (gemini | openai | anthropic)
LLM_PROVIDER=gemini

# LLM Keys
GEMINI_API_KEY=your_gemini_api_key
GEMINI_MODEL=gemini-1.5-pro

OPENAI_API_KEY=your_openai_api_key
OPENAI_MODEL=gpt-4o

ANTHROPIC_API_KEY=your_anthropic_api_key
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022

# OMDb API Key (for metadata fetching)
OMDB_API_KEY=your_omdb_api_key

```

### Running the Admin Tools

From the `admin-tools` directory:

1.  **Backfill Episode Metadata (CLI)**:
    Fetch episode details (directors, writers, plot, IMDb ratings, etc.) from OMDb API and insert them into the database:
    ```bash
    python omdb_fetch.py --season 1 --episode 4 --episode-id s1e4
    ```

2.  **Generate a Review Draft from a Transcript (CLI)**:
    Feed a transcript file to the LLM to get a structured review JSON draft:
    ```bash
    python generate_review.py --transcript transcript.txt --episode-id s1e4 --provider gemini --out s1e4_review.json
    ```

3.  **Upload generated reviews to Supabase (CLI)**:
    ```bash
    python supabase_upsert.py --review-json s1e4_review.json
    ```

4.  **Run the Streamlit GUI App**:
    An all-in-one local interface to upload transcripts, select episodes, preview/edit LLM-generated JSON reviews, and push them to Supabase:
    ```bash
    streamlit run app.py
    ```
